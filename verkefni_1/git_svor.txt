#Git spurningar og svör

1. Hvað gera eftirfarandi Linux skipanir?
	cd		= breytir um möppu sem þú ert staðsettur í
	cd ~		= fer beint í heimasvæði notanda
	ls		= sýnir alla hluti inni í möppunni sem þú ert staðsettur í
	pwd		= sýnir í hvaða möppu þú ert 
	mkdir		= býr til möppu

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone	= klónar repository á local svæði
	git log		= sýnir öll commit-in
	git status	= sýnir statusinn á verkefninu, commitum og svoleiðis
	git diff	= sýnir breytingar á milli committum
	git checkout	= breyta á milli greina eða endurheimta skrár

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff 			= sýnir breytingar á committum og því sem hefur verið breytt
	b) git diff --staged 		= sýnir muninn á því sem þú ert búinn að gera og því sem þú ert ekki búinn að cista/committa
	c) git diff commit1 commit2	= ber saman og sýnir muninn á tveiumr ákveðnum committum

4. 	Hvað er útgáfustýring (e.version control)? 
					= hugbúnaður sem sér um að halda utan um mismunandi útgáfur og breytingar á verkefnum

5.	Hverjir eru helstu kostir við að nota GIT?
					= > að geta deilt verkefninu með öðru fólki án vesens
					  > að margir geti unnið að verkefninu og breytt hlutum on the go, eða hvar sem er
					  > það heldur utan um allar breytingar og hægt er að leiðrétta mistök

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?
					= Þegar hefur verið gert patch, lítið fix eða bara við allar breytingar sem skipta máli

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
					= Working directory 	> Staðurinn sem er tengdur við repositoryið, sem sagt local mappan þar sem allt er
					  Staging area 		> skrá sem heldur utan um breytingar þangað til þú committar þær
					  Repository 		> Repository er staðurinn þar sem skrárnar eru geymdar á github
